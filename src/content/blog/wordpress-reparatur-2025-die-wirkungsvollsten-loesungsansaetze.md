            <h2 class="text-3xl font-bold mb-4 text-gray-900">Schritt 1: Sofortmaßnahmen für die WordPress Reparatur</h2>
            
            <div class="warning-box">
                <p><i class="fas fa-exclamation-triangle mr-2"></i><strong>Wichtig:</strong> Erstellen Sie vor jeder Reparatur ein vollständiges Backup Ihrer Website. Dies ist die wichtigste Sicherheitsmaßnahme bei jeder WordPress Reparatur.</p>
            </div>
            
            <h3 class="text-2xl font-semibold mb-3">Backup erstellen</h3>
            <p class="mb-4">Nutzen Sie eines der folgenden bewährten Backup-Plugins:</p>
            
            <ul class="list-disc pl-6 mb-6">
                <li><strong>UpdraftPlus:</strong> Kostenlose Version für grundlegende Backups</li>
                <li><strong>BackWPup:</strong> Umfassende Backup-Lösung mit Scheduler</li>
                <li><strong>Duplicator:</strong> Ideal für Website-Migrationen</li>
            </ul>

            <h3 class="text-2xl font-semibold mb-3">WordPress im Safe Mode starten</h3>
            <p class="mb-4">Deaktivieren Sie alle Plugins über die Datenbank oder per FTP, um Plugin-Konflikte auszuschließen:</p>
            
            <div class="code-block">
// In der wp-config.php hinzufügen:
define('WP_DEBUG', true);
define('WP_DEBUG_LOG', true);
define('WP_DEBUG_DISPLAY', false);

// Alle Plugins deaktivieren - Ordner umbenennen
plugins/ → plugins-deactivated/
            </div>

            <h2 class="text-3xl font-bold mb-4 text-gray-900">Schritt 2: Systematische Problemdiagnose</h2>
            
            <p class="mb-4">Eine erfolgreiche WordPress Reparatur beginnt mit einer gründlichen Diagnose. Folgen Sie diesem bewährten Workflow:</p>

            <h3 class="text-2xl font-semibold mb-3">Error-Logs auswerten</h3>
            <p class="mb-4">Aktivieren Sie das WordPress-Debugging und analysieren Sie die Fehlermeldungen:</p>
            
            <div class="code-block">
// wp-config.php - Debug-Einstellungen
ini_set('log_errors', 1);
ini_set('error_log', '/path/to/wp-content/debug.log');
define('WP_DEBUG', true);
define('WP_DEBUG_LOG', true);
define('SCRIPT_DEBUG', true);
            </div>

            <div class="success-box">
                <p><i class="fas fa-lightbulb mr-2"></i><strong>Profi-Tipp:</strong> Die meisten Hosting-Provider bieten Error-Logs im Control Panel an. Schauen Sie dort zuerst nach konkreten Fehlermeldungen.</p>
            </div>

            <h3 class="text-2xl font-semibold mb-3">Plugin-Konflikte identifizieren</h3>
            <p class="mb-4">Gehen Sie systematisch vor:</p>
            
            <ol class="list-decimal pl-6 mb-6">
                <li>Alle Plugins deaktivieren</li>
                <li>Website testen - funktioniert sie wieder?</li>
                <li>Plugins einzeln reaktivieren</li>
                <li>Nach jeder Aktivierung testen</li>
                <li>Problematisches Plugin identifizieren</li>
            </ol>

            <h2 class="text-3xl font-bold mb-4 text-gray-900">Schritt 3: Spezifische Reparatur-Maßnahmen</h2>

            <h3 class="text-2xl font-semibold mb-3">White Screen of Death beheben</h3>
            <p class="mb-4">Der gefürchtete weiße Bildschirm ist meist durch Speicher-Probleme oder PHP-Fehler verursacht. So lösen Sie das Problem:</p>

            <div class="code-block">
// 1. Speicher-Limit erhöhen in wp-config.php
ini_set('memory_limit', '256M');

// 2. Oder in .htaccess
php_value memory_limit 256M

// 3. Maximum execution time erhöhen
ini_set('max_execution_time', 300);
            </div>

            <h3 class="text-2xl font-semibold mb-3">Datenbank-Reparatur</h3>
            <p class="mb-4">WordPress bietet eingebaute Tools für die Datenbank-Reparatur:</p>

            <div class="code-block">
// wp-config.php - Datenbank-Reparatur aktivieren
define('WP_ALLOW_REPAIR', true);

// Dann aufrufen: www.ihredomain.de/wp-admin/maint/repair.php
// Nach der Reparatur: define('WP_ALLOW_REPAIR', false);
            </div>

    <script>
        function toggleAccordion(element) {
            const content = element.nextElementSibling;
            const icon = element.querySelector('i');
            
            // Close all other accordion items
            document.querySelectorAll('.accordion-content').forEach(item => {
                if (item !== content) {
                    item.classList.remove('active');
                    item.previousElementSibling.classList.remove('active');
                    item.previousElementSibling.querySelector('i').classList.replace('fa-chevron-up', 'fa-chevron-down');
                }
            });
            
            // Toggle current accordion item
            content.classList.toggle('active');
            element.classList.toggle('active');
            
            // Toggle icon
            if (content.classList.contains('active')) {
                icon.classList.replace('fa-chevron-down', 'fa-chevron-up');
            } else {
                icon.classList.replace('fa-chevron-up', 'fa-chevron-down');
            }
        }
    </script>
